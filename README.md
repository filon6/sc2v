
# sc2v (исправленная сборка)

Форк инструмента **sc2v** — переводчика RTL‑моделей SystemC в Verilog.

Оригинальный репозиторий плохо собирается на современных системах (Fedora, Ubuntu и др.) из-за устаревшего кода и ошибок в Makefile.  
В этом форке исправлена система сборки, поэтому инструмент можно скомпилировать и использовать на современных версиях GCC, Flex и Bison.

---

## Что было исправлено

В оригинальной версии возникали следующие проблемы:

- проект не собирался на современных версиях GCC
- ошибки в Makefile
- предупреждения компилятора превращались в ошибки
- несовместимость с новыми версиями `flex` и `bison`

В этом форке исправлены Makefile и параметры компиляции, поэтому теперь корректно собираются все этапы транслятора:

```
sc2v_step1
sc2v_step2
sc2v_step3
```

---

## Требования

Перед сборкой необходимо установить зависимости.

### Fedora

```bash
sudo dnf install flex bison gcc gcc-c++ glibc-devel
```

### Ubuntu / Debian

```bash
sudo apt install flex bison build-essential
```

---

## Сборка

Клонировать репозиторий и собрать проект:

```bash
git clone https://github.com/<ваш-username>/sc2v.git
cd sc2v/src
make clean
make
```

После сборки в папке `bin` появятся файлы:

```
sc2v_step1
sc2v_step2
sc2v_step3
```

---

## Использование

Поместите RTL‑файлы SystemC (`.cpp` и `.h`) в папку `bin`, затем выполните:

```bash
cd bin
./sc2v.sh <имя_модуля>
```

Пример:

```
./sc2v.sh ImpulseGenRTL
```

После выполнения будет создан файл Verilog:

```
ImpulseGenRTL.v
```

---

## Ограничения

Транслятор довольно старый и поддерживает только ограниченное подмножество SystemC.

Для корректной трансляции рекомендуется:

- не использовать `#ifndef / #define` в заголовочных файлах
- избегать `clk.pos()`
- использовать простые `SC_METHOD`
